Data Formats

1) Pass 1 (assessments/pass1_individual/*.json)
- JSON structure:
  - assessor_id (string)
  - role (string)
  - rubric_points_possible (number)
  - scores (list)
    - student_id (string)
    - rubric_total_points (number)
    - criteria_points (object, optional)
    - notes (string)

2) Pass 2 (assessments/pass2_comparative/*)
- Plain text list
- One student_id per line (best -> worst)

3) Conventions Report (processing/conventions_report.csv)
- Generated by scripts/conventions_scan.py
- Key field: student_id

4) Consensus Output (outputs/consensus_scores.csv)
- Includes rubric mean, conventions rate, Borda points, composite score, flags

5) Final Pairwise Review (assessments/final_review_pairs.json)
- Adjacent pairs with keep/swap decisions and reasons

6) Final Order (outputs/final_order.csv)
- Consensus rows with `final_rank` after pairwise review

6b) Final Review Log (outputs/final_review_log.md)
- Human-readable keep/swap decisions with reasons

6c) Flagged Pairwise Decisions (outputs/final_review_flagged.md)
- Swaps below confidence threshold for manual review

9) LLM Pairwise Input (assessments/final_review_llm_input.json)
- Outline + adjacent pairs with essay text for LLM decision support

10) LLM Pairwise Output (assessments/final_review_llm_output.json)
- Same schema as input; includes filled decision.action, decision.reason, decision.confidence

11) Usage Log (outputs/usage_log.jsonl)
- One JSON per line with model and token usage

12) Usage Cost Report (outputs/usage_costs.json)
- Aggregated model costs based on pricing config

13) Cost Limits (config/cost_limits.json)
- Per-student, per-job, and per-call limits for LLM usage

7) Dashboard Data (outputs/dashboard_data.json)
- UI payload with student text and round summaries

8) Class Metadata (inputs/class_metadata.json)
- Optional: {"grade_level": 7, "class_name": "..."}
