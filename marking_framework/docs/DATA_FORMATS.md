# Data Formats

1) Pass 1 (`assessments/pass1_individual/*.json`)
- JSON structure:
  - assessor_id (string)
  - role (string)
  - rubric_points_possible (number)
  - scores (list)
    - student_id (string)
    - rubric_total_points (number)
    - criteria_points (object, optional)
    - notes (string)

2) Pass 2 (`assessments/pass2_comparative/*`)
- Plain text list
- One student_id per line (best -> worst)

3) Conventions Report (`processing/conventions_report.csv`)
- Generated by `scripts/conventions_scan.py`
- Key field: student_id

4) Consensus Output (`outputs/consensus_scores.csv`)
- Includes rubric mean, conventions rate, Borda points, composite score, flags

5) Pairwise Consistency Checks (`outputs/consistency_checks.json`)
- Generated by `scripts/verify_consistency.py`
- Contains cheap/orientation pairwise judgments with pair, seed order, winner, decision, confidence, decision basis, cautions, criterion notes, and model metadata

6) Escalated Pairwise Checks
- `outputs/pairwise_escalation_candidates.json`
  - high-leverage candidate pairs and trigger reasons
- `outputs/pairwise_escalations.json`
  - stronger routed adjudications
- `outputs/consistency_checks.escalated.json`
  - merged cheap/orientation/escalated judgments with source metadata

7) Committee-Edge Resolver Artifacts
- `outputs/committee_edge_candidates.json`
  - residual unstable edges selected for committee routing
- `outputs/committee_edge_decisions.json`
  - emitted committee-edge decisions, if any
- `outputs/committee_edge_report.json`
  - trigger, budget, guard, and passthrough summary
- `outputs/committee_edge_live_trace.json`
  - live/fixture read trace when committee reads are enabled
- `outputs/consistency_checks.committee_edge.json`
  - canonical merged judgment file consumed by `global_rerank.py`; surviving direct `adjudication_source="committee_edge"` judgments are eligible for protected `committee_direct_edge` rerank constraints

8) Evidence Map And Group Packets
- `outputs/evidence_map.json`
  - deterministic claim/evidence/commentary map per student
- `outputs/evidence_neighborhood_report.json`
  - local placement-neighborhood diagnostics
- `outputs/evidence_group_calibration_packets.json`
  - bounded packets for group calibration reads

9) Pairwise Adjudicator Eval
- `outputs/pairwise_adjudicator_eval.json`
  - hard-pair accuracy, critical accuracy, coverage, and polish-bias risk report

10) Final Order (`outputs/final_order.csv`)
- Consensus rows with `final_rank` after deterministic global rerank.
- Related diagnostics live in `outputs/consistency_report.json`, including `committee_direct_edge_*` counts when committee-edge decisions are protected, satisfied, violated, or dropped for graph safety.

11) Dashboard Data (`outputs/dashboard_data.json`)
- UI payload with student text and round summaries.

12) Class Metadata (`inputs/class_metadata.json`)
- Optional: `{"grade_level": 7, "class_name": "..."}`

13) Legacy Optional Pairwise Review
- `assessments/final_review_pairs.json`
- `outputs/final_review_log.md`
- `outputs/final_review_flagged.md`
- `assessments/final_review_llm_input.json`
- `assessments/final_review_llm_output.json`
- These older swap-review artifacts still exist for manual review workflows, but the canonical rank path is now consistency checks plus escalation, committee-edge merge, and global rerank.

14) Usage Log (`outputs/usage_log.jsonl`)
- One JSON per line with model and token usage

15) Usage Cost Report (`outputs/usage_costs.json`)
- Aggregated model costs based on pricing config

16) Cost Limits (`config/cost_limits.json`)
- Per-student, per-job, and per-call limits for LLM usage
